## Overview

DishDash is a data-driven restaurant discovery platform focused on the NYU/Greenwich Village neighborhood. The system integrates geographic, ratings, nutritional, and sentiment data into a unified SQLite database, enabling rich analytics and (planned) a web-based search interface.

---

## Module Map

### Notebooks

| File | Primary Function |
|------|-----------------|
| `DishDash_Milestone3-5.ipynb` | End-to-end data pipeline: fetches data from 3 APIs, merges into a unified DataFrame, runs sentiment analysis, and stores everything in SQLite. Contains all backend data collection logic for Milestones 3–5. |

### Python Modules / Functions (within the notebook)

| Function / Section | Location | Purpose |
|--------------------|----------|---------|
| `fetch_osm_restaurants()` | Cell 7 | Queries the Overpass API (OpenStreetMap) for restaurants, cafes, and fast food within a bounding box around NYU. Returns a cleaned DataFrame. |
| `fetch_google_places()` | Cell 10 | Calls the Google Places API (New) Nearby Search endpoint using a POST request with a FieldMask. Returns ratings, reviews, price levels, and types. |
| `fetch_nutrition_for_cuisine()` | Cell 15 | Queries the Open Food Facts API for nutritional data by food search term, aggregates results by cuisine category. |
| `haversine_meters()` | Cell 18 | Computes geographic distance in meters between two lat/lng coordinate pairs using the Haversine formula. |
| `fuzzy_geo_merge()` | Cell 19 | LEFT JOINs OSM and Google Places data using combined geographic proximity (≤75m) and fuzzy name matching (token sort ratio ≥55). |
| `normalize_cuisine()` | Cell 21 | Maps raw OSM cuisine tags to a standardized key matching the nutrition lookup table. Handles aliases and multi-value tags. |
| `analyze_sentiment_hf()` | Cell 24 | Calls the HuggingFace Inference API with DistilBERT (SST-2) to classify Google review text as POSITIVE or NEGATIVE. |
| SQLite storage section | Cells 28–31 | Writes three tables (`venues`, `nutrition_profiles`, `data_sources`) to `dishdash.db` and verifies row counts with sample queries. |

### Data Files

| File | Description |
|------|-------------|
| `dishdash.db` | SQLite database containing all merged venue, nutrition, and metadata records. Generated by running the notebook. |

## Data Flow Summary

```
Overpass API (OSM)    ──┐
                         ├── fuzzy_geo_merge() ──> df_venues
Google Places API    ──┘
                                  │
                                  ├── normalize_cuisine() ──> cuisine_key
                                  │
Open Food Facts      ─────────────┴──> LEFT JOIN on cuisine_key ──> df_final
                                                    │
HuggingFace NLP      ──────────────────────────────┘ (sentiment on reviews)
                                                    │
                                               SQLite DB
                                         (venues, nutrition_profiles,
                                          data_sources tables)
```
