# 🍜 DishDash — Web App

Hyperlocal restaurant discovery for the NYU / Greenwich Village area.  
Built with Streamlit — runs locally in seconds and deploys for free.

---

## File Structure

```
dishdash_app/
├── app.py            ← Streamlit web app (the frontend)
├── pipeline.py       ← Data pipeline (OSM + Google + Open Food Facts → SQLite)
├── requirements.txt  ← Python dependencies
└── README.md
```

`dishdash.db` is generated by the pipeline and read by the app.  
It is **not** committed to git (add it to `.gitignore`).

---

## Quickstart (local)

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Run the data pipeline
This fetches live data from all three APIs and creates `dishdash.db`.
```bash
python pipeline.py
```
> **If your API keys change**, update `GOOGLE_API_KEY` and `HUGGINGFACE_TOKEN`  
> at the top of `pipeline.py`, or set them as environment variables:
> ```bash
> export GOOGLE_API_KEY="your_key_here"
> export HUGGINGFACE_TOKEN="your_token_here"
> python pipeline.py
> ```
> The app always works even without `dishdash.db` — it falls back to a  
> built-in 30-venue demo dataset automatically.

### 3. Launch the app
```bash
streamlit run app.py
```
Opens at **http://localhost:8501** in your browser.

## Features

| Tab | What it does |
|-----|-------------|
| 🔍 Search | Filter by name, cuisine, price, rating, Nutri-Score — results update live |
| 🗺️ Map | Interactive Folium map, colour-coded by rating, clickable popups |
| 📊 Analytics | 4-panel dashboard: rating histogram, cuisine pie, calorie chart, sentiment |
| 🤖 Recommender | Plain-English query → top 3 venue picks with explanation |

All filters in the **sidebar** apply across Search and Map simultaneously.

## Deploy to Streamlit Cloud (free, public URL)

1. Push this folder to a **GitHub repository**:
   ```bash
   git init
   git add app.py pipeline.py requirements.txt README.md
   git commit -m "Initial DishDash app"
   git remote add origin https://github.com/YOUR_USERNAME/dishdash.git
   git push -u origin main
   ```

2. Go to **[share.streamlit.io](https://share.streamlit.io)** → Sign in with GitHub

3. Click **New app** → select your repo → set **Main file path** to `app.py`

4. Add your API keys under **Advanced settings → Secrets**:
   ```toml
   GOOGLE_API_KEY = "your_key_here"
   HUGGINGFACE_TOKEN = "your_token_here"
   ```

5. Click **Deploy** — you get a public URL like `https://dishdash.streamlit.app`

> **Note on the database:** Streamlit Cloud doesn't persist files between  
> restarts. For production, either commit a pre-built `dishdash.db` to the  
> repo (fine for a class project), or migrate to a cloud database like  
> [Supabase](https://supabase.com) (free tier) for persistent storage.

---

## Refreshing the data

Re-run `python pipeline.py` any time to update `dishdash.db` with fresh  
API data. The Streamlit app caches data for 5 minutes (`@st.cache_data(ttl=300)`),  
so a page refresh after the pipeline finishes will pick up new results.
